Compile targets

One source skill compiles to eight targets. Lazy targets keep a stub out of context until invoked; eager targets carry the whole body every session — kitbash measures that cost instead of hiding it.

What eager loading costs

A skill authored to lazy-load cannot lazy-load on an eager target — its whole body sits in the context window every session. compile surfaces that on every eager target:

⚠ prereview → agentsmd: agentsmd is eager and cannot lazy-load;
  this skill costs ~1,480 tokens standing every session (declared limit: 60)

The measured benchmark puts the lazy-vs-eager gap at roughly 12× per session for the reference skill.

Honest degradation

Adapters declare capabilities (claude-code: scripts, hooks, subagents; most others: none yet). A skill's targets.requires is crossed with those to derive its compatibility matrix — a target missing a capability gets an instruction-only build with a visible warning. kitbash explain <skill> <adapter> prints the exact reason.
