Commands

What the CLI does today. Anything not listed as working prints its roadmap milestone and exits — no silent stubs.

Core loop

kitbash init

Sets up the repo: writes kitbash.toml (targets + [policy]) and .kitbash/skills/.

kitbash install <source> [--yes]

Sources: gh:owner/repo[/path][@ref], bare owner/repo, or file:path. Prints a review block first — permissions (network/write), budget, standing cost, capability requirements, lint warnings — then prompts install? [y/N] on a TTY. --yes skips the prompt; [policy] rules are enforced either way. Plain SKILL.md-only folders (the skills.sh / Claude Skills convention) install directly and are flagged unmanifested.

kitbash compile [--strict]

Emits every detected target's native format. Warns on budget overruns and eager-loading costs; --strict turns warnings into failures. Recompiles are idempotent; outputs of removed skills are pruned. Only files bearing the generated header are ever touched.

kitbash remove <skill> · kitbash list

Remove drops the skill and its lock pin (then compile prunes its outputs); list shows installed skills with versions and context cost.

Inspection

All three accept an installed skill name, a directory path, or an uninstalled remote source (gh:owner/repo/path) — so you can read a skill before it touches your repo.

kitbash preview <skill|path|source>

Renders each configured adapter's compiled output with per-target token counts and loading mode.

kitbash lint [skill|path|source] [--strict]

The full static check suite: schema, context budgets, dead references, injection heuristics. No argument lints everything installed.

kitbash explain <skill|path|source> <adapter>

Why a compilation degrades on that adapter — which required capability is missing — plus the standing token cost when a lazy skill lands on an eager target.

Health

kitbash doctor

Detects assistants, reports total standing context cost, verifies kitbash.lock integrity (exit 1 on drift), and rechecks [policy] against installed skills.

kitbash test [skill] [--strict]

Runs a skill's static-tier evals: schema, measured budgets, dead refs, injection heuristics.
