Skill format

A skill is a directory: a skill.toml manifest plus a SKILL.md instruction body. The manifest declares things no other agent format has a field for — an enforced token budget and tool permissions. Full authority: the spec (KSF).

The manifest

# skill.toml — [skill] and [context] are required; everything else defaults restrictive
[skill]
name = "prereview"
version = "0.3.1"              # semver — breaking prompt changes are breaking changes
description = "Review a diff against this team's real standards"

[triggers]
commands = ["/prereview"]      # compiled to native slash commands where supported
events = ["pre-push"]          # hook points, for gate-mode skills

[context]
budget = 1500                  # REQUIRED. max tokens injected when active — enforced at compile
standing = 80                  # max tokens present when NOT invoked
disclosure = "lazy"            # lazy | eager

[permissions]
tools = ["read", "grep", "bash:git *"]
network = false
write = false

[artifacts]
produces = ["findings@1"]
consumes = ["plan@1"]          # optional inputs; the skill must work without them

[targets]
requires = []                  # capabilities: scripts | hooks | subagents | network
mode = "skill"                 # skill | gate

What the key fields buy you

  • context.budget / standing — compile measures the emitted output against these and warns or fails (--strict). The hidden token tax becomes a number.
  • permissions — surfaced in the install review and compiled to native permission rules where the target supports them; advisory elsewhere, honestly labeled.
  • targets.requires — crossed with each adapter's capabilities to derive the compatibility matrix. An agent that can't run scripts gets an instruction-only build with a visible warning — never a quiet downgrade. kitbash explain shows exactly why.
  • artifacts — typed inputs/outputs so skills pipe into each other through data instead of prompt-chaining hope.

Already have plain SKILL.md skills?

A bare SKILL.md folder — the skills.sh / Claude Skills convention — installs directly with kitbash install owner/repo. A manifest is synthesized with conservative defaults and the skill is flagged unmanifested at install, list, and compile. You keep your skills; you gain a lockfile and a token-cost report.
