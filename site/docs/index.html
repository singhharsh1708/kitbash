Documentation

Kitbash is an open format for AI agent skills (KSF) and a compiler that turns one source folder into whatever each coding agent actually reads — v0.6.0, zero runtime dependencies, Apache-2.0.

Quickstart

# in your repo
npm install -g kitbash
kitbash init
kitbash install gh:owner/repo/path/to/skill
kitbash compile

That's the whole loop. init writes kitbash.toml, install reviews and pins a skill, compile emits each detected agent's native format — .claude/skills/, .cursor/rules/, AGENTS.md sections, and five more. Compiled output is plain files in your repo: no runtime, no lock-in.

Why it exists

Every agent invented its own instruction format, so teams hand-maintain divergent copies that quietly drift. Kitbash treats a skill as an engineering artifact: written once, compiled per agent, with three properties no hand-maintained copy has:

  • Measured token budgets — every skill declares its context cost; compile measures the real output and warns (or fails with --strict) when it goes over.
  • Pinned integrity — installs pin a content hash in kitbash.lock; doctor detects drift.
  • Reviewable trust — permissions, budget, and lint findings are shown before anything is written, and org [policy] rules gate installs.
